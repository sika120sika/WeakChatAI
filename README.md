# WeakChatAI ⚡

Ollama で動くローカル LLM とブラウザからチャットできる Web アプリ。コーディングアシスタント用途に特化。

- **ビルド不要・Node.js 不要** — `index.html` を開くだけで通常チャットが動く
- **完全ローカル** — データは一切外部に送信されない
- **エージェントモード** — `server.py` を起動するとファイル操作・コマンド実行が可能

## 必要環境

| 用途 | 必要なもの |
|---|---|
| 通常チャット | Ollama + ブラウザ |
| エージェントモード | 上記 + Python 3.x |

動作確認環境: Windows 11 / GTX 3070 (VRAM 8GB) / RAM 32GB

## セットアップ

### 1. Ollama のインストール

[https://ollama.ai](https://ollama.ai) からインストール後、CORS を許可する。

```powershell
[System.Environment]::SetEnvironmentVariable("OLLAMA_ORIGINS", "*", "User")
```

システムトレイの Ollama を **Quit → 再起動**。`ollama serve` は不要（トレイアプリがサーバーを兼ねる）。

### 2. モデルのインストール

アプリ内の **「📥 モデルDL」** ボタンから GUI でダウンロードできる。

推奨: **qwen2.5-coder:7b**（4.7 GB・VRAM 8GB 以内・コーディング特化）

### 3. 起動

**通常チャット**: `index.html` をブラウザで開く

**エージェントモード**:
```powershell
python server.py
```
→ ブラウザで `index.html` を開き **「🤖 エージェント」** ボタンをクリック

### エージェントモード（server.py 必要）

`python server.py` で起動するローカルツールサーバー（port 3000）。ブラウザから直接ファイル操作・コマンド実行が可能になる。

#### エージェントツール

| ツール | 承認 | 説明 |
|---|---|---|
| `read_file` | 自動 | ファイル内容を読み取り |
| `list_dir` | 自動 | ディレクトリ一覧表示 |
| `search_content` | 自動 | コードファイル内テキスト検索（最大 50 件） |
| `find_files` | 自動 | ファイル名パターン検索（`*.py`, `*config*` など、最大 200 件） |
| `write_file` | **要承認** | ファイル作成・上書き（内容プレビュー付き） |
| `run_command` | **要承認** | シェルコマンド実行（タイムアウト 30 秒） |

#### セキュリティ

- **作業ディレクトリ外アクセス禁止** — パストラバーサル攻撃をサーバー側でブロック
- **承認フロー** — `write_file` / `run_command` はユーザー確認ダイアログが必須
- **危険コマンド二重確認** — `rm -rf`, `git reset --hard`, `git push --force` 等は 2 クリック必要
- **承認キャンセル** — 「↺ リセット」ボタンで待機中の承認ダイアログを一括キャンセル

#### エージェント特有の機能

- **プロジェクトツリー自動注入** — エージェント起動時にファイル構造をシステムプロンプトへ自動追加。AI が最初からプロジェクト全体を把握できるため、ファイルを手探りする無駄がなくなる
- **ツールコールフォールバック** — 構造化 tool_calls が返らない場合、レスポンスのコードブロック内 JSON を解析して実行
- **最大 15 ステップ** — エージェントループの上限で無限実行を防止

## Ollama API リファレンス

| エンドポイント | メソッド | 用途 |
|---|---|---|
| `/api/tags` | GET | インストール済みモデル一覧 |
| `/api/show` | POST | モデル情報取得（コンテキスト長など） |
| `/api/chat` | POST | チャット（messages 配列・stream 対応） |
| `/api/generate` | POST | テキスト生成（prompt 文字列・stream 対応） |
| `/api/pull` | POST | モデルダウンロード（NDJSON 進捗ストリーム） |
| `/api/delete` | DELETE | モデル削除 |
